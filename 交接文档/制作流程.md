## 1. 环境安装（第一步，一次装好）

整套依赖三块：**XeLaTeX 引擎 + 中英字体**（排版）、**poppler**（把 PDF 渲成图）、**Python 读图库**（裁剪/放大截图）。

```bash
# ① LaTeX 引擎 + 宏包(tcolorbox/fancyhdr/enumitem/ulem/tabularx/tikz 等)
sudo apt-get update
sudo apt-get install -y texlive-xetex texlive-latex-extra texlive-pictures

# ② 字体：中文 Noto CJK + 拉丁 TeX Gyre Heros(Helvetica 风)
sudo apt-get install -y fonts-noto-cjk fonts-texgyre

# ③ PDF 工具(pdfinfo / pdftoppm / pdftotext)
sudo apt-get install -y poppler-utils

# ④ Python 读图依赖(脚本 render.py / zoom.py / crop_tool.py 用)
python3 -c "import fitz,PIL,numpy" 2>/dev/null || pip install PyMuPDF Pillow numpy
```

**字体清单**（`ptestyle.sty` 里写死要用）：`Noto Sans CJK SC`、`Noto Serif CJK SC`、`TeX Gyre Heros`。

**装完自检**：

```bash
xelatex --version | head -1                       # 有版本号即可
fc-list | grep -iE "Noto Sans CJK SC|TeX Gyre Heros" | head   # 能列出即字体在
pdftoppm -v 2>&1 | head -1                         # poppler 在
python3 -c "import fitz,PIL,numpy; print('py ok')"  # 读图库在
```

> 克隆 + 开分支：`git clone <仓库网址> && cd <目录> && git checkout -b <任务指定的分支>`。

## 2. 三条铁律（最重要）

1. **截图 PDF 没有文字层 → 必须渲染成图片用眼睛读。** `pdftotext` / `build/extract/*.txt` 基本是空的，**别依赖文字提取**；一切靠渲成 PNG、放大、肉眼逐字誊录。
2. **一切以原图为准，人工对照誊录。** 分数、每个批改词、AI 中文建议、平台配色，都要和截图核对一致；拿不准就放大再看。**平台用了颜色，你就要复刻颜色**（别只写文字摘要）。
3. **成品版本化、旧版永不改。** 成品放唯一的 `整理卷/`，**每个 part 各自从 `000` 起** +1；纯文字版本单文件自包含，含图片版本做成子文件夹。详见 §4 / §A。

│   ├─ 写作/   ← 000/001/002 的 .pdf+.tex   纯文字版本 = 松散两文件
│   ├─ 阅读/   ← 000/001
│   └─ 听力/   ← 000/001
│
└─ build/                          ← 【工作区/工具】可重新生成
    ├─ render.py zoom.py pngcrop.py crop_tool.py   读图工具(渲染/放大/裁剪截图)

    └─ paper/   ← 在这里排版
        ├─ ptestyle.sty   共享样式(配色 + 所有宏)，**别重写**
        ├─ cover.tex      封面(成绩总览)
        ├─ sec_<part>_body.tex   各部分正文(writing/speaking/reading/listening)
        ├─ preview_<part>.tex    预览主文件(\input cover + body)
        └─ img/  (临时图片工作目录，.gitignore 内)

> **`原卷素材/` = 只读输入**；**`build/paper/` = 可编辑工作台**；**`整理卷/` = 冻结成品**
> （根 = 4 个最新成品 PDF，子文件夹 = 各 part 全部版本存档）。

**命名规则**：版本化成品文件名一律 `<序号>_PTE模考<编号>_<部分>_整理卷`（`.pdf` + 同名 `.tex`）。
- `<序号>`：三位，**每个 part 各自从 `000` 起**，每更新一版 +1（写作 000/001/002…、口语 000/001…，互不干扰）。某 part 最新版 = 它名字下最大的序号。
- `<编号>`：本次模考编号（看成绩图，如 `1B`）。
- `<部分>`：`写作 / 口语 / 阅读 / 听力`。

**两层存放（务必照做）**：
1. **版本化存档**进各 part 子文件夹：`整理卷/<部分>/<序号>_…_整理卷.{pdf,tex}`（含图片版本则是子文件夹，见下）。`flatten.py` 已直接输出到这里。
2. **最终成品**（每 part 最新版的 PDF）复制一份到 `整理卷/` **根目录**，改成**干净名** `PTE模考<编号>_<部分>_整理卷.pdf`（无序号）。根目录**只放这 4 个 PDF**，给人直接取用；出新版本后记得更新根目录这份。

**纯文字 vs 含图片**：写作/阅读/听力无图 → 松散 `.pdf`+`.tex` 放 `整理卷/<部分>/`；口语 DI 有图 → 做成**子文件夹** `整理卷/口语/<序号>_…_整理卷/`，内含 `.tex` + `img/` + `.pdf`，「tex+图片」整体自包含。

**旧版本只读、永不修改/覆盖**，要改只新增下一个序号。

## 5. 通用流程（六步）

1. **读懂截图**（§6 技巧）：`pdfinfo` 看页数尺寸 → `pdftoppm` 整本渲图 → 逐页建版面地图 → `zoom.py` 放大读小字。
2. **在 `build/paper/` 排版**：复用 `ptestyle.sty`（§7 工具箱）；新部分新建 `sec_<part>_body.tex` + `preview_<part>.tex`；封面 `cover.tex` 改成绩数字（条长 = 得分/90）。
3. **逐题誊录**：题干/原文、范文、你的作答、逐项评分、彩色批改、AI 建议——**一字一句对照原图**。**每题加中文翻译**（`\zh{…}`）。具体要素见**对应分册**。
4. **编译 + 保真核对**：`cd build/paper && xelatex preview_<part>.tex`（跑两遍）→ 把成品渲成 PNG 和原图并排比对 → 改到一致。原图分页截断/缺内容，**照实标注红字，不要瞎补**。
5. **出版本**：`build/flatten.py` 展平到 `整理卷/`（§8），编译验证与 preview 一致。
6. **提交推送**（§9）。

## 6. 读图技巧

```bash
pdfinfo 口语详解1.pdf | grep -iE "pages|page size"     # 常是 A3，一页叠几张手机截图
pdftoppm -png -r 130 口语详解1.pdf /tmp/d              # 整本渲成 /tmp/d-1.png …（逐页 Read 看）

# 放大某一小块读小字(分数/AI建议/彩色词)：坐标是页面比例 0~1
python3 build/zoom.py 口语详解1.pdf <0基页号> x0 y0 x1 y1 <dpi> <名字>   # → build/zoom/<名字>.png
python3 build/zoom.py 口语详解1.pdf 0 0.30 0.10 0.70 0.30 400 q1_score

python3 build/render.py    口语1.pdf 0          # 自动裁灰底卡片+切长图 → build/pages/
python3 build/crop_tool.py 口语1.pdf 0 350 /tmp/c.png   # 自动紧贴内容裁一刀
```

> A3 详解页常把多张手机截图竖排，且**卡片下方显示的「原文/题干」往往属于下一题**（截图滚动错位）——按**内容**配对，别按位置。详见口语分册。

## 7. 共享 LaTeX 工具箱（`ptestyle.sty`，别重写）

**批改 / 标注宏**

| 宏 | 含义 | 外观 |
|----|------|------|
| `\fix{原}{正}` | 修改 / 拼写 / 缺空格 | 红删除线 + 绿改正 |
| `\del{词}` | 应删除 | 蓝删除线 + “删” |
| `\ins{词}` | 应插入 | 紫色 + “∧插” |
| `\add{原}{正}` | **平台漏标、本卷补标** | 橙波浪线 + “补” |
| `\miss{词}` | 漏读 / 未作答（灰） | 灰删除线 |

**口语 AI 语音识别配色**（复刻平台逐词评测）

| 宏 | 含义 |
|----|------|
| `\sg{词}` `\sy{词}` `\sr{词}` | 绿=优 / 橙=良 / 红=差 |
| `\smiss{词}` | 漏读/未读准（红删除线） |
| `\pse` | 停顿（灰 `/`） |
| `\asrlegend` | 一行图例（优/良/差/停顿/漏读） |

**评分 / 翻译**

| 宏 | 含义 |
|----|------|
| `\sfull{x/y}` `\spart{x/y}` `\szero{x/y}` | 评分：满分绿 / 部分橙 / 零低红 |
| `\zh{中文}` | 中文翻译行（蓝“译” + 灰译文）——**每题都加** |

**结构件**

- `\ptesection{中文}{英文}{该项分}` 章节大标题；`\ptetask{中文}{英文}` 题型小节分隔；`\ptesub{小标题}`。
- 题块 `\begin{qblock}{标号·题型}{右上:得分/用时} … \end{qblock}`。
- 文本框：`promptbox`（题干/原文，灰）、`modelbox`（范文，绿）、`youbox`（你的作答，红）、`notebox`（注解，蓝）、`asrbox`（语音识别，白）。
- 评分表：`tabularx` + `\scorehead`（用法照抄 `sec_writing_body.tex`）。
- 新增宏请加在 `ptestyle.sty`（只增不改既有定义，以免破坏旧版的展平副本）。

## 8. 出版本（`build/flatten.py`）

脚本读取 `ptestyle.sty + cover.tex + sec_<part>_body.tex`，内联展平成自包含成品，**输出到 `整理卷/<部分>/`**。

```bash
ls 整理卷/<部分>/                            # 看该 part 现有最大序号，本次用下一个

# 纯文字: python3 build/flatten.py <part英文> <中文> <编号> <序号>
python3 build/flatten.py reading 阅读 1B 002        # → 整理卷/阅读/002_PTE模考1B_阅读_整理卷.tex

# 含图片: 末尾多传图片目录名(图片放 build/paper/<目录>/，正文 \includegraphics{<目录>/x.png})
python3 build/flatten.py speaking 口语 1B 001 img   # → 整理卷/口语/001_…口语…/(含 tex+img/)

# 到目标目录编译两遍，清掉中间文件
cd "整理卷/口语/001_PTE模考1B_口语_整理卷"   # 纯文字版本则 cd 整理卷/<部分>/
xelatex 001_PTE模考1B_口语_整理卷.tex && xelatex 001_PTE模考1B_口语_整理卷.tex
rm -f *.aux *.log *.out

# ★ 若这是该 part 最新版：把 PDF 复制到 整理卷/ 根并改成干净名（根目录只放 4 个最终成品）
cp 001_PTE模考1B_口语_整理卷.pdf ../../PTE模考1B_口语_整理卷.pdf
```

**验证**：`diff <(pdftotext 成品.pdf -) <(pdftotext build/paper/preview_<part>.pdf -)` 应一致。

### 8.1 合订本（四部分合并 + 封面 + 目录）

四部分都做完后，用 `build/flatten_all.py` 合成一本【总整理卷】（封面 + 目录 + 听·说·读·写）：

```bash
python3 build/flatten_all.py 1B            # → 整理卷/总整理卷/PTE模考1B_总整理卷.tex（+ img/）
cd 整理卷/总整理卷
xelatex "PTE模考1B_总整理卷.tex" && xelatex "PTE模考1B_总整理卷.tex"   # 跑两遍, 目录页码才对
rm -f *.aux *.log *.out *.toc
cp PTE模考1B_总整理卷.pdf ../PTE模考1B_总整理卷.pdf   # 合订本 PDF 放 整理卷/ 根(最显眼)
```

- 目录靠 `ptestyle.sty` 里 `\ptesection`/`\ptetask` 自动写入（`\addcontentsline`），无需手动列。
- 预览版是 `build/paper/preview_all.tex`（cover + `\tableofcontents` + 概览表 + 四部分 `\input`）。
- 含口语 DI 图 → 合订本是「子文件夹 + img/」的含图片版本。

## A. 版本规则（复述，必须照做）

1. 一个版本 = 一对成品 `<序号>_PTE模考<编号>_<部分>_整理卷` 的 `.pdf` + `.tex`。
2. **版本化存档**按 part 分文件夹：`整理卷/{口语,写作,阅读,听力}/`，每个 part 所有版本都进自己的子文件夹。
3. **最终成品**（每 part 最新版 PDF）另复制一份到 `整理卷/` 根，干净名 `PTE模考<编号>_<部分>_整理卷.pdf`；根目录**只放这 4 个**。
4. **每个 part 各自从 `000` 起 +1**（三位数，互不干扰）。
5. **旧版本只读，永不修改/覆盖**；只新增下一个序号。
6. 纯文字版本 = 单文件自包含 `.tex` 放 `整理卷/<部分>/`；含图片版本 = 子文件夹（tex+img/+pdf）。都用 `build/flatten.py`（已输出到 `整理卷/<部分>/`）。
7. **原始素材**统一放 `原卷素材/`（只读）；**交接文档**统一放 `交接文档/`。

## B. 命令速查

```bash
pdfinfo X.pdf                                      # 页数/尺寸
pdftoppm -png -r 130 X.pdf /tmp/x                  # 整本渲染成 PNG
python3 build/zoom.py X.pdf 0 x0 y0 x1 y1 400 名     # 按比例放大一块(读小字)
python3 build/render.py X.pdf 0                    # 自动裁卡片+切片 → build/pages/
python3 build/crop_tool.py X.pdf 0 350 /tmp/c.png  # 紧裁内容
cd build/paper && xelatex preview_<part>.tex       # 预览(跑两遍)
python3 build/flatten.py <part> <中文> <编号> <序号> [img]   # 出版本
```
