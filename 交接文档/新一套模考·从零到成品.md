## 0. 这是什么（30 秒）

把 **APEUni / 猩际 PTE Core 模考**的「答案 & 评分详情」截图（原图不可复制、只能看图）**逐题人工誊写、重新排版**成干净可打印的整理卷 PDF。四个 part：**听力 / 阅读 / 口语 / 写作**，各自成卷，最后可合订成一本【总整理卷】。

**三条铁律**：
1. **截图无文字层 → 一切渲成图片用眼睛读、人工逐字誊录**（别指望 `pdftotext`/文字提取）。
2. **忠于原图、复刻平台颜色**（平台用色就逐处还原；平台漏标的错用橙色「补」标）。
3. **成品版本化、旧版永不改**（每个 part 各自从 `000` 起 +1）。

## 1. 环境安装（第一步，约 2–4 分钟，需联网）

> ⚠️ 这是**实测可用**的命令（root 下去掉 `sudo`）。旧版总册曾漏装 `texlive-latex-recommended / texlive-plain-generic / texlive-lang-chinese`，那样会**编译失败**——以本篇为准。

```bash
apt-get update
apt-get install -y \
  texlive-xetex texlive-latex-recommended texlive-latex-extra \
  texlive-plain-generic texlive-lang-chinese \
  fonts-noto-cjk fonts-texgyre poppler-utils
pip3 install Pillow
# 仅当本套素材是「PDF 题目卷 + PDF 详解卷」(需要 PDF 渲染/裁剪工具) 时，额外装：
pip3 install PyMuPDF numpy
```

**每个包为什么需要**（对照 `build/paper/ptestyle.sty` 的 `\usepackage`）：

| 需要的东西 | 来自哪个包 |
|------|------|
| `xelatex` 引擎、`fancyhdr`/`tabularx`/`pifont`/`graphicx` | `texlive-xetex`（含 latex-base）|
| `fontspec`、`ragged2e` | `texlive-latex-recommended` |
| `tcolorbox`、`enumitem`、`needspace`、`tikz` | `texlive-latex-extra`（连带 texlive-pictures）|
| `ulem`（删除线 `\sout`）| `texlive-plain-generic` |
| `xeCJK`（中文排版）| `texlive-lang-chinese`（连带 texlive-lang-cjk）|
| 字体 `Noto Sans CJK SC` / `Noto Serif CJK SC` | `fonts-noto-cjk` |
| 字体 `TeX Gyre Heros`（拉丁，Helvetica 风）| `fonts-texgyre` |
| `pdfinfo` / `pdftoppm`（把成品 PDF 渲成 PNG 自检）| `poppler-utils` |
| `Pillow`（裁 PNG/JPEG 长截图、DI 配图）| pip：所有套都要 |
| `PyMuPDF`(`fitz`)+`numpy`（驱动 `zoom.py`/`render.py`/`crop_tool.py`）| pip：**只有 PDF 素材的套才要** |

**装完自检**（四行都要有输出）：

```bash
xelatex --version | head -1
fc-list | grep -iE "Noto (Sans|Serif) CJK SC|TeX Gyre Heros" | sort -u
pdftoppm -v 2>&1 | head -1
python3 -c "import PIL; print('Pillow', PIL.__version__)"
```

> 字体名在 `ptestyle.sty` 里**写死**（`\setmainfont{TeX Gyre Heros}`、`\setCJKmainfont{Noto Sans CJK SC}`、`Noto Serif CJK SC`）。换环境先 `fc-list` 确认装上，否则编译报缺字体。

**克隆 + 开分支**：`git clone <新仓库网址> && cd <目录>`，切到**任务指定的分支**（如 `git checkout -b <分支名>`）。所有改动只推这条分支，**不要自动开 PR**（除非明确要求）。

## 2. 认素材 + 定参数

1. 把模考导出的**所有截图/文件**放进 `原卷素材/`（**只读**）。
2. 看 `成绩.png`，记下：
   - **模考编号**（如 `1B`/`2B`/`3A`…）→ 命名里的 `<编号>`。
   - **提交时间** + **四项分数 + 总分** → 封面 `cover.tex` 的数字来源（**每套必改**）。
3. 判断**输入形态**（决定用哪套读图工具）：
   - **PDF 题目卷 + PDF 详解卷**（如 1B）→ 用 `build/zoom.py` / `render.py` / `crop_tool.py`（**需 PyMuPDF**）。
   - **docx 内嵌截图 + PNG/JPEG 长截图**（如 2B）→ docx 先导出内嵌图：
     `unzip -o -j 原卷素材/X详解.docx 'word/media/*' -d /tmp/x`，再 `Read` 逐张看；长 PNG 用 `build/pngcrop.py` 或直接 Pillow 裁（**只需 Pillow**）。
4. **黄金法则**：**题目 / 配图 / 范文 / 选项 / ASQ 答案** 去「题目卷（列表视图）/ 总览图」找；**分数 + 彩色批改 + AI 建议** 去「详解（docx / 详解卷）」找。配图、范文常在题目卷里清晰可见，详解卷里反而被弹窗盖住。

## 3. 一道题属于哪个 part（认标题前缀）

复盘截图每题开头是 `N. 题型 #ID 标题`，认前缀即可定位：

| 前缀 | part | 前缀 | part |
|------|------|------|------|
| **SWT** 概括写作 · **WE** 邮件 | ✍️ 写作 | **SST** 概述口语 | 🎧 听力 |
| **RA** 朗读 · **RS** 复述 · **DI** 描述图 · **RTS** 情景应答 · **ASQ** 简答 | 🗣️ 口语 | **MCM-L** / **MCS-L** / **FIB-L** | 🎧 听力 |
| **FIB**（下拉选词）· **FIB(R)**（拖词）· **MCM-R** / **MCS-R** · **RO** 段落重排 | 📖 阅读 | **SMW** 选词 · **HIW** 找错词 · **WFD** 听写句子 | 🎧 听力 |

**易混**：`-R`=阅读、`-L`=听力；`SST`(听·Spoken) ≠ `SWT`(写·Written)；`RO` 只在阅读；`RA/RS/DI/RTS/ASQ` 只在口语；`SMW/HIW/WFD` 只在听力。完整对照见 `00_总交接文档`。

## 4. 排版（在 `build/paper/`）

- 复用 `build/paper/ptestyle.sty`（所有配色 + 宏）。**别重写**；要加宏只往后追加、**不改既有定义**（否则破坏旧版展平副本）。
- 每个 part 编辑 `sec_<part>_body.tex`（`writing`/`reading`/`listening`/`speaking`），预览主文件 `preview_<part>.tex`（`\input cover.tex` + `\input sec_<part>_body.tex`）。
- 封面 `cover.tex`：把**总分、四项分数、条形图条长**（条长 = 得分/90）改成本套数字。
- 逐题要素：题干/原文/录音原文、范文/正确答案、你的作答、逐项评分、彩色批改、AI 建议，**+ 每题中文翻译 `\zh{}`**。每个 part 的具体结构看**对应分册**。
- 编译预览（**跑两遍**，目录/引用第二遍才对）：
  ```bash
  cd build/paper && xelatex -interaction=nonstopmode preview_<part>.tex && xelatex -interaction=nonstopmode preview_<part>.tex
  ```
- **保真自检**：`pdftoppm -png -r 110 preview_<part>.pdf /tmp/p` → 逐页 `Read` PNG，和原图并排比对配色/分数/标注，改到一致。原图分页截断/缺内容，**照实标红字说明，别瞎补**。

**`ptestyle.sty` 宏速查**：
- 结构：`\ptesection{中}{英}{该项分}`、`\ptetask{中}{英}`、`\ptesub{小标题}`、`qblock`；文本框 `promptbox`(题干灰)/`modelbox`(范文绿)/`youbox`(作答红)/`notebox`(注解蓝)/`asrbox`(语音识别白)。
- 批改：`\fix{原}{正}`(改/拼写)、`\del`(删)、`\ins`(插)、`\add{原}{正}`(平台漏标·本卷补标)、`\miss`(漏读灰)。
- 阅读：`\rok{词}`(选对✓)、`\rno{你的}{答案}`(选错✗)、`\rblank{答案}`(直接给空)。
- 听力：HIW `\hiw{屏幕错词}{实际读音}`；WFD `\lhit`(命中绿)/`\womit`(漏写红括号)/`\wbad`(多写/听错灰删除线) + `\wfdlegend`。⚠️ WFD 配色各套平台可能不同，**忠于原图**。
- 口语 AI 语音识别：`\sg`绿优 / `\sy`橙良 / `\sr`红差 / `\smiss`红删除线漏读 / `\pse`停顿 / `\asrlegend`图例；配色文本放 `asrbox`。
- 评分：`\sfull`(满分绿)/`\spart`(部分橙)/`\szero`(零低红)；翻译 `\zh{中文}`（**每题都加**）。

## 5. 命名 & 版本规则（务必照做）

- 成品名：**`<序号>_PTE模考<编号>_<部分>_整理卷`**（`.pdf` + 同名 `.tex`）。
  - `<序号>`：三位数，**每个 part 各自从 `000` 起**，每更新一版 +1（写作 000/001… 与口语 000/001… 互不干扰）。
  - `<部分>`：`写作` / `口语` / `阅读` / `听力`。
- **两层存放**：
  1. **版本存档** → `整理卷/<部分>/`（`flatten.py` 已直接输出到这里）。
  2. **每 part 最新版 PDF** 复制一份到 `整理卷/` **根**，改**干净名** `PTE模考<编号>_<部分>_整理卷.pdf`。**根目录只放最终成品 PDF**（4 个 part + 合订本）。
- **纯文字版本**（写作/阅读/听力，无图）= 松散 `.tex` + `.pdf`。
  **含图版本**（口语 DI 有配图）= **子文件夹** `整理卷/口语/<序号>_…/`（内含 `.tex` + `img/` + `.pdf`，整体自包含）。
- **旧版本只读、永不改/覆盖**；要改只 +1 出新序号。原始素材放 `原卷素材/`（只读），文档放 `交接文档/`。

## 6. 出版本 + 合订本（`build/flatten.py` / `flatten_all.py`）

```bash
# ① 出某 part 版本（脚本把 ptestyle.sty + cover.tex + sec_<part>_body.tex 展平成自包含成品）
#    含图的 part(口语) 末尾多传一个图片目录名 img（图片放 build/paper/img/，正文用 \includegraphics{img/x.png}）
python3 build/flatten.py <part英文> <中文> <编号> <序号> [img]
#    例：python3 build/flatten.py reading 阅读 2B 000
#        python3 build/flatten.py speaking 口语 2B 000 img

# ② 到生成目录编译两遍、清中间文件
cd "<生成目录>"     # 纯文字: 整理卷/<部分>/   含图: 整理卷/口语/<序号>_…/
xelatex "<文件名>.tex" && xelatex "<文件名>.tex" && rm -f *.aux *.log *.out *.toc

# ③ 若是该 part 最新版：复制成干净名到 整理卷/ 根
cp "<文件名>.pdf" "../../PTE模考<编号>_<部分>_整理卷.pdf"   # 路径按实际层级调整

# ④ 四部分齐 → 合订本（封面 + 目录 + 听·说·读·写）
python3 build/flatten_all.py <编号>
cd 整理卷/总整理卷 && xelatex "PTE模考<编号>_总整理卷.tex" && xelatex "PTE模考<编号>_总整理卷.tex" && rm -f *.aux *.log *.out *.toc
cp "PTE模考<编号>_总整理卷.pdf" "../PTE模考<编号>_总整理卷.pdf"
```

> ⚠️ **`build/flatten_all.py` 里的「本卷概览」表（四项分数 + 总分）是写死的**，不会随 `cover.tex` 自动变。**出新一套时务必把 `toc_overview` 里的四项分数和总分改成本套数字**（`cover.tex` 同样要改）。本套 2B 已是 47/36/46/71、总分 50。
> 目录页码靠 `\ptesection`/`\ptetask` 的 `\addcontentsline` 自动生成，所以合订本要**跑两遍** xelatex。

## 8. 踩过的坑（真实经验，照避省时间）

- **图片阅读配额**：超长 PNG / 大量 docx 内嵌图**逐张 `Read` 会读爆配额**。原则：**能用 Pillow 裁就别整张 Read**（Pillow 裁切不占配额，只有 `Read` 看图才占）。DI 配图**先按估计坐标裁出小图再 Read 核对边界、微调**，别整张读原图。
- **`\zh{}` 里出现 `%` 必须转义成 `\%`**：中文译文写百分比（如「约 42%」）若不转义，`%` 会把整行注释掉、导致 `}` 失配编译错。范文/题干里的 `% & # _ $` 同理要转义。
- **DI 配图裁切一定要渲图核对**：估计坐标常切掉图的一边或带进 UI（导航条 / 题号 / 范文文字）。裁完 `Read` 小图确认四边干净、主体完整，再定稿。宽图 `\includegraphics` 用 0.82~0.9，双图 0.95~0.97。
- **数据量大时程序化生成更稳**：例如口语 24 条「逐词配色串」，与其手抄不如把原始串（`词(g) 词(y) /pse …`）**程序解析后机械转成 `\sg{}\sy{}\pse`**，并做 **token 数自检**（原串词数 == 生成的宏数），避免手抄错漏；英文原文也尽量从交接文档程序化取，杜绝转写误差。
- **每题都要 `\zh` 翻译**；只写文字摘要不算数，**配色要逐处复刻**；平台漏标的错用 `\add` 橙色补标。
- **编译跑两遍**（目录/引用），合订本尤其；编完 `rm -f *.aux *.log *.out *.toc`，根目录只留干净名 PDF。
- **分支**：只推任务指定的分支；若交接文档里写的分支名与本次会话被配置的分支不一致，**先和用户确认**再推。
